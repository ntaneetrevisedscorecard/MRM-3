One-Time Setup — Connect Google Sheets

1

Create Google Sheet

Go to sheets.google.com → New blank spreadsheet → Name it "MRH Hospital"

2

Open Apps Script

Click menu: Extensions → Apps Script. A new tab opens. Delete any existing code there.

3

Paste Backend Script

Open the file google-apps-script.js that came with your website. Copy all of it. Paste into Apps Script editor. On line 3, change the email to hospital's email. Press Ctrl+S to save.

4

Deploy as Web App

Click Deploy → New Deployment. Type: Web App. Execute as: Me. Access: Anyone. Click Deploy. Authorise when prompted. Copy the URL shown — you need this in steps 5 and 6.

5

Update index.html

Open index.html in Notepad. Press Ctrl+F, find https://script.google.com/macros/s/AKfycbzY1y2blxtTjzsdQnd3G01-gQ7i9wboJjDV9kcTdTpn9hqc_3nsgy7xyeADt2AfPKuQ/exec. Replace it with the URL you copied. Save the file.

6

Update admin-portal.html

Open admin-portal.html in Notepad. Find https://script.google.com/macros/s/AKfycbzY1y2blxtTjzsdQnd3G01-gQ7i9wboJjDV9kcTdTpn9hqc_3nsgy7xyeADt2AfPKuQ/exec. Replace with the same URL. Save.

7

Change Admin Password

In admin-portal.html, find hospital@2025. Change to your own password. Save.

8

Upload & Test

Upload all 3 files to your hosting (Hostinger / Netlify / any host). Open the website, make a test booking — it should appear in Google Sheets within seconds and you get an email notification at the hospital's email.

9

Test the Admin Portal

Login to admin-portal.html. Go to Tests & Prices and change any price. Open the website in another tab — the price should update immediately without touching any code.

🔒 Security — Is The Website Safe?

✅ Google Sheets backend — Data lives in Google's servers. No one can delete or edit it unless they have access to your Google account.

✅ Admin password protection — No one can access this portal without the password. Change it anytime in the HTML file.

✅ Razorpay payments — All payment processing is done by Razorpay's secure servers. Card details never touch your website.

⚠️ Script URL exposure — The Google Apps Script URL is visible in the HTML source. This is unavoidable for a client-side website. To prevent abuse: go to your Apps Script → Add a check for a secret token in doPost/doGet. Your developer can add this in 10 minutes if needed.

⚠️ No HTTPS = Risk — Make sure your hosting uses HTTPS (the padlock icon). All good hosts (Hostinger, Netlify) do this free automatically.

❌ Not suitable for — storing patient medical records, HIPAA-level data, or sensitive clinical information. This website is for bookings and information only — not a medical records system.
